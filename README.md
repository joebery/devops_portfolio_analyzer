# DevOps Portfolio Analyzer\n\nA cloud-native SaaS application that analyzes public GitHub repositories using AI to generate professional project summaries, DevOps maturity scores, and improvement suggestions for README files.\n\n![Python](https://img.shields.io/badge/language-Python-blue.svg) ![License](https://img.shields.io/badge/license-MIT-green.svg) ![Docker](https://img.shields.io/badge/docker-enabled-lightblue.svg)\n\n---\n\n## 🚀 What This Project Does\n\nPaste a public GitHub URL. The app:\n\n1. Fetches repository metadata via the GitHub API.\n2. Scans the repo structure for DevOps signals — Dockerfiles, Terraform, GitHub Actions workflows, Kubernetes manifests, and more.\n3. Sends the findings to an AI model.\n4. Returns:\n   - A plain English project summary.\n   - A DevOps maturity score.\n   - README improvement suggestions.\n\nBuilt to demonstrate real-world cloud engineering patterns — not a tutorial app.\n\n---\n\n## 🛠️ Tech Stack\n\n| Layer | Technology |\n|---|---|\n| Frontend | Next.js |\n| Backend | Python / FastAPI |\n| Database | PostgreSQL |\n| Queue | AWS SQS |\n| Storage | AWS S3 |\n| AI | OpenAI API |\n| Infrastructure | AWS (ECS Fargate, RDS, S3, SQS, ECR) |\n| Containers | Docker / Docker Compose |\n\n---\n\n## ✨ Features\n\n- 🚀 Automated GitHub repository analysis\n- 📊 AI-generated project summaries and DevOps maturity scores\n- 🛠️ Suggestions for README file improvements\n\n---\n\n## 🚧 Getting Started\n\n### Prerequisites\n\n- [Docker Architecture\n\n```\nUser submits GitHub URL\n        ↓\nFastAPI validates and saves job (status: pending)\n        ↓\nJob dropped onto SQS queue\n        ↓\nBackground worker picks up job:\n  → Calls GitHub API for metadata\n  → Scans repo file tree for DevOps signals\n  → Sends structured data to OpenAI\n  → Stores AI analysis in PostgreSQL + S3\n        ↓\nFrontend polls for job completion\n        ↓\nResults dashboard rendered\n```\n\nThe API and worker are deliberately decoupled. The API responds instantly with a job ID — the slow work (GitHub + OpenAI calls) happens in the background. This prevents users from waiting on a synchronous chain of external API calls.\n\n---\n\n## 🤝 Contributing\n\nContributions are welcome!
